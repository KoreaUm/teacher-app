# 쌤포트 기능과 목적 정리

작성일: 2026-05-05  
프로그램명: 쌤포트  
형태: 교사용 Electron 데스크톱 앱  
주요 대상: 담임교사, 상담교사, 특성화고/상업고 취업지도 담당 교사

## 1. 프로그램의 목적

쌤포트는 교사의 학급 운영, 학생 기록, 상담, 출결, 평가, 일정, 공문, 취업지도 업무를 한 곳에서 관리하기 위한 교사용 업무 관리 프로그램이다.

교사가 매일 처리하는 업무는 여러 곳에 흩어져 있다. 학생 명단은 엑셀에 있고, 상담 기록은 별도 문서에 있으며, 할일은 메신저나 공문 속에 남아 있고, 시간표와 급식, 학사일정은 NEIS나 학교 홈페이지를 따로 확인해야 한다. 쌤포트는 이런 업무를 하나의 데스크톱 앱 안에 모아 교사가 빠르게 확인하고 기록하고 다시 활용할 수 있도록 돕는다.

특히 학생 상담, 출결, 성적, 내신, 자격증, 취업 가능성처럼 민감한 정보는 외부 서버로 보내지 않는 것을 기본 원칙으로 한다. 학생별 데이터는 로컬 PC에 저장하고, AI 기능도 가능한 경우 로컬 AI를 우선 사용하도록 설계한다.

## 2. 핵심 방향

- 교사가 매일 확인해야 하는 정보를 대시보드에서 한눈에 볼 수 있게 한다.
- 학생별 기록을 흩어지지 않게 누적하고, 상담과 생활지도에 다시 활용할 수 있게 한다.
- 반복적인 행정 문장 작성, 할일 정리, 시간표 구조화 같은 업무를 AI로 줄인다.
- 학생 정보, 상담 내용, 성적관리 데이터는 원칙적으로 로컬에 저장한다.
- 클라우드 AI를 사용할 때는 외부 전송 가능성을 분명히 안내하고 민감정보 입력을 제한한다.
- 취업 추천 데이터는 공식 채용 공고, ALIO, 공식 PDF처럼 검증 가능한 출처만 사용한다.

## 3. 개인정보와 데이터 저장 원칙

쌤포트에서 가장 중요한 원칙은 학생 정보 보호다.

- 학생 명단, 상담 기록, 출결, 관찰 기록, 성적관리 데이터, 내신, 자격증, 학생별 취업 분석 결과는 로컬 DB에 저장한다.
- 성적관리 데이터는 클라우드로 동기화하지 않는다.
- 바로가기 링크나 공식 채용 공고 데이터처럼 학생 개인정보가 아닌 공용 정보는 공유 가능 데이터로 볼 수 있다.
- 학생 실명, 상담 내용, 성적, 내신, 보호자 정보가 포함될 수 있는 내용은 외부 AI로 보내지 않는 것을 원칙으로 한다.
- 로컬 AI를 선택하면 Ollama 기반 모델이 사용자 PC 안에서만 실행된다.
- 클라우드 AI를 선택할 경우 입력 내용이 외부 AI 서버로 전송될 수 있으므로 사용자의 명시적 주의와 동의가 필요하다.

## 5. 대시보드

대시보드는 쌤포트의 시작 화면이다. 교사가 앱을 열었을 때 가장 먼저 확인해야 할 정보를 카드형 위젯으로 보여준다.

주요 기능:

- 오늘 날짜, 달력, 날씨, 시계 확인
- 오늘의 할일 확인
- 할일 추가, 수정, 완료, 삭제
- 지난 할일 정리
- 학사일정 확인
- 급식 메뉴 확인
- D-Day 관리
- 개인 시간표 확인
- 학급 시간표 확인
- 연락처와 바로가기 관리
- AI 할일 추출
- 위젯 배열 편집과 위치/크기 조정

활용 예시:

- 아침에 앱을 열고 오늘 수업, 급식, 일정, 미완료 할일을 확인한다.
- 공문이나 메신저 내용을 붙여넣고 AI 할일 추출로 업무 목록을 만든다.
- 자주 쓰는 위젯은 위쪽에 배치해 매일 확인 시간을 줄인다.

## 6. 학생 명단

학생 명단은 다른 기록 기능의 기준이 되는 핵심 데이터다.

주요 기능:

- 학생 번호, 이름, 성별, 생년월일, 연락처, 보호자 연락처, 주소, 특이사항 저장
- 학생 추가, 수정, 삭제
- CSV 가져오기
- 상담, 출결, 관찰, 평가 페이지에서 학생 선택 기준으로 사용

운영 원칙:

- 학기 초에 학생 명단을 먼저 정리하면 이후 상담, 출결, 평가 기록이 안정적으로 연결된다.
- 학생 개인정보이므로 외부 공유나 클라우드 전송에 주의해야 한다.

## 7. 출결 관리

출결 관리는 날짜별 학생 상태를 기록하는 기능이다.

주요 기능:

- 날짜별 출결 입력
- 출석, 결석, 지각, 조퇴 등 상태 관리
- 출결 사유와 비고 기록
- 월별 출결 통계 확인

활용 예시:

- 상담 전 특정 학생의 결석, 지각 변화가 있는지 확인한다.
- 생활지도나 보호자 상담 시 객관적 근거 자료로 활용한다.

## 8. 학급 메모

학급 메모는 날짜별로 학급 운영 상황을 간단히 남기는 공간이다.

주요 기능:

- 하루 동안 기억해야 할 학급 상황 기록
- 전달 사항, 특이사항, 생활지도 포인트 기록
- 추후 상담 기록이나 학기 말 정리 자료로 활용

활용 예시:

- “체육대회 연습 중 갈등 발생”, “학급 회의에서 나온 건의사항”처럼 짧은 기록을 남긴다.
- 나중에 상담이나 생활기록 자료를 정리할 때 날짜별 맥락을 확인한다.

## 9. 시간표 관리

시간표 관리는 개인 시간표와 학급 시간표를 관리하는 기능이다.

주요 기능:

- 요일/교시별 개인 시간표 입력
- 시간표 텍스트를 AI로 구조화
- 시간표 사진 분석 기능 지원
- 설정에서 학급 시간표 엑셀 업로드
- 대시보드에 개인 시간표와 학급 시간표 표시
- 학급 시간표 과목별 색상 설정

주의사항:

- 텍스트 기반 시간표 분석은 로컬 AI로도 처리할 수 있다.
- 시간표 사진 분석은 이미지 인식이 가능한 클라우드 AI가 필요할 수 있다.

## 10. 상담 일지

상담 일지는 학생 상담을 단순 기록으로 끝내지 않고 후속 조치까지 이어지도록 설계된 기능이다.

주요 기능:

- 상담 날짜, 학생, 교사 역할, 상담 분야 기록
- 상담 유형, 주제, 요약, 상세 내용 기록
- 위험도, 위험 신호, 상담 상태 기록
- 후속 예정일과 다음 조치 기록
- 상담 기록 검색
- 상담 기록 CSV 내보내기
- AI 도우미를 통한 상담 문장, 후속 조치, 요청 메시지 작성 보조

상담 기록 원칙:

- 학생 발화, 교사 관찰, 합의한 행동, 후속 확인일을 구분해 기록한다.
- 위험도가 높거나 자해, 폭력, 가정 위기 신호가 있으면 앱 기록만으로 끝내지 않고 학교 절차에 따라 관리자, 전문상담교사, 보호자 연계를 검토한다.
- AI 답변은 참고 자료이며 최종 판단은 교사가 실제 맥락과 학교 절차에 따라 해야 한다.

## 11. 관찰 기록

관찰 기록은 수업과 생활 속 학생 행동을 누적하는 기능이다.

주요 기능:

- 학생별 관찰 내용 저장
- 날짜, 교과, 상황, 행동 기록
- 긍정 행동, 변화, 지도 필요 사항 기록
- 상담과 생활지도 근거로 활용

활용 원칙:

- 평가적 표현보다 관찰 가능한 사실 중심으로 기록한다.
- “태도가 나쁨”보다 “수업 시작 후 15분 동안 과제지를 제출하지 않고 엎드려 있었음”처럼 구체적으로 적는다.

## 12. 수업 지도

수업 지도 기능은 수업 운영 기록과 수업 개선 메모를 남기는 공간이다.

주요 기능:

- 수업 내용 기록
- 학생 반응 기록
- 보완할 점과 다음 수업 준비 사항 기록
- 추후 평가, 상담, 수업 개선 자료로 활용

## 13. 수행평가

수행평가는 평가 항목과 학생별 점수를 관리하는 기능이다.

주요 기능:

- 평가명, 교과, 유형, 날짜, 만점 설정
- 학생별 점수 입력
- 평가 결과 저장
- 통계와 학생 분석에 활용

운영 팁:

- 평가 기준과 실시 날짜를 함께 남겨두면 점수 확인과 이의 신청 대응이 쉬워진다.

## 14. 제출물 관리

제출물 관리는 과제와 제출 여부를 학생별로 확인하는 기능이다.

주요 기능:

- 제출물 이름, 교과, 마감일, 비고 등록
- 학생별 제출 여부 체크
- 미제출 학생 확인

활용 예시:

- 마감일이 있는 제출물은 대시보드 할일에도 함께 등록해 누락을 줄인다.

## 15. 통계와 출력

통계와 출력은 저장된 기록을 요약하고 업무 확인용 자료로 정리하는 기능이다.

주요 기능:

- 출결, 평가, 기록 데이터 요약
- 학생별 기록 확인
- 보고나 출력 자료 준비 보조

## 16. AI 분석

AI 분석은 저장된 자료를 바탕으로 교사의 판단을 돕는 보조 기능이다.

주요 기능:

- 학생 정보, 출결, 상담, 관찰 기록을 바탕으로 상담/관찰 포인트 정리
- 로컬 AI 기반 분석 지원
- 후속 조치와 생활지도 관점 정리

중요 원칙:

- 학생 정보가 포함되는 분석은 로컬 AI 사용을 우선한다.
- AI 분석 결과는 최종 판단이 아니라 교사를 돕는 참고 자료다.
- 민감한 학생 정보를 클라우드 AI로 보내지 않는다.

## 17. 급식 메뉴

급식 메뉴는 NEIS 급식 정보를 불러와 날짜별 식단을 확인하는 기능이다.

주요 기능:

- 교육청 코드와 학교 코드를 기준으로 급식 조회
- 조식, 중식, 석식 확인
- 열량 정보 표시
- 대시보드 급식 위젯과 연동

## 18. 학사 일정

학사 일정은 NEIS 학사일정을 조회하고 학교 주요 일정을 확인하는 기능이다.

주요 기능:

- 월별 학교 일정 조회
- 대시보드 일정 위젯과 연동
- 행사, 시험, 방학, 재량휴업일 등 확인

활용 예시:

- 행사 전 준비물이 필요한 일정은 할일로 함께 등록한다.

## 19. 계산기

계산기는 앱 안에서 간단한 산술 계산을 처리하는 보조 도구다.

활용 예시:

- 성적 비율 계산
- 예산 합계 계산
- 간단한 업무 숫자 확인

## 20. 공문서 작성

공문서 작성 기능은 학교 행정 문서 작성을 돕기 위한 기능이다.

주요 기능:

- 공문 초안 작성
- 공문 문장 검토
- 공공언어 기준에 맞춘 제목, 관련, 본문, 붙임, 끝. 형식 확인
- 공문 템플릿 활용
- 견적서 이미지나 PDF에서 품목 정보 추출
- AI를 활용한 공문 초안 생성 지원

주의사항:

- 공문에 학생 개인정보나 민감정보가 포함되면 외부 AI 사용 전에 반드시 익명화하거나 로컬 AI 사용을 우선한다.
- AI가 만든 공문은 최종 제출 전 교사가 문서 형식과 사실관계를 확인해야 한다.

## 21. 할일 목록

할일 목록은 교사가 해야 할 업무를 기한과 우선순위 중심으로 관리하는 기능이다.

주요 기능:

- 할일 제목, 마감일, 우선순위, 카테고리 저장
- 완료 처리
- 수정과 삭제
- 대시보드에서 빠른 확인
- AI 할일 추출 결과 자동 추가
- Google 캘린더 연동 설정 시 캘린더 일정 등록 지원

활용 예시:

- 공문, 문자, 카카오톡 안내 내용을 붙여넣어 AI가 할일만 추출하게 한다.
- 마감일이 있는 업무는 대시보드에서 매일 확인한다.

## 22. 설정

설정은 앱의 기본 동작과 개인 환경을 관리하는 영역이다.

주요 기능:

- 학년, 반, 교사명 설정
- 학교 코드, 교육청 코드 설정
- 날씨 지역 설정
- 상단 메뉴 구성 조정
- 대시보드 바로가기 관리
- 학급 시간표 엑셀 업로드
- 학급 시간표 과목별 색상 설정
- 로컬 AI와 클라우드 AI 선택
- API 키 관리
- Ollama 상태 확인과 로컬 AI 자동 설치/확인

## 23. AI 도우미

AI 도우미는 모든 페이지에서 열 수 있는 업무 보조 채팅이다.

주요 기능:

- 현재 페이지의 기능 설명
- 앱 사용법 안내
- 상담 후속 조치 문장 작성
- 보호자, 상담교사, 학생에게 보낼 공손한 메시지 작성
- 공문 문장 정리
- 할일 정리
- 로컬 AI 또는 클라우드 AI 기반 답변

응답 원칙:

- 답변은 선생님을 대상으로 한다.
- 학생을 사용자로 오해하지 않는다.
- 매크로처럼 같은 문장을 반복하지 않고 질문 맥락에 맞게 구체적으로 답한다.
- 상담, 성적, 개인정보와 관련된 질문은 로컬 AI 사용을 우선한다.
- 공식 설명서 내용을 참고해 앱 기능 질문에 답할 수 있다.

## 24. 성적 관리와 취업 데이터 관리

성적 관리 페이지는 단순 점수 기록을 넘어, 특성화고/상업고 학생의 취업 가능성을 분석하기 위한 로컬 데이터 관리 영역이다.

주요 기능:

- 현학생/졸업생 성적 데이터 관리
- 내신과 평균 내신 기록
- 출결 요약 기록
- 자격증 보유 현황 기록
- 희망 회사와 희망 직무 기록
- 취업 회사와 취업 직무 기록
- 지역과 비고 기록
- 성적관리 CSV 가져오기/내보내기
- 성적관리 로컬 초기화
- 로컬 성적 컬럼과 점수 저장
- 학생별 조건으로 취업 가능성 분석

취업 데이터 기능:

- 공공기관/공기업 고졸 채용 공식 데이터 조회
- 금융권 고졸/특성화고 채용 데이터 조회
- 최근 5년 기준 연도 탭 제공
- 금융권만 보기
- 검증된 가산점만 보기
- 전체 보기
- 제한전형 포함 여부 선택
- 기관/기업명 검색
- 자격증/분야 검색
- 공식 출처 버튼 제공

공식 출처 원칙:

- 기관 공식 채용 홈페이지
- ALIO 공공기관 채용정보
- 기관이 직접 게시한 PDF 채용공고
- NCS 등 공적 채용 공고 페이지
- 공공기관 채용정보박람회 공식 자료

사용하지 않는 출처:

- 블로그
- 카페
- 취업 커뮤니티
- 요약 기사
- 학원 자료
- 출처가 불명확한 채용 요약
- 공고문이 아닌 기관 메인 홈페이지
- 채용 목록만 있는 페이지

데이터 상태값:

- 검증됨: 공식 원문 근거가 있고 추천 근거로 표시 가능
- 부분확인: 일부 조건은 확인되지만 점수, 단계, 적용 범위 등이 불완전함
- 확인불가: 공식 원문으로 확인하지 못함
- 만료/과거공고: 과거 기준으로 보관하며 현재 추천 점수에 직접 반영하지 않음

추천 로직 주의사항:

- 자격증 가산점과 응시자격은 반드시 구분한다.
- 응시자격을 가산점처럼 더하지 않는다.
- 과거 공고를 현재 지원 가능한 공고처럼 보여주지 않는다.
- 장애인, 보훈, 제한경쟁, 체험형 인턴, 정규직 고졸 전형은 서로 다른 트랙으로 구분한다.
- 학생별 내신, 출결, 자격증 데이터는 로컬에서만 비교한다.

## 25. 백업과 이동

쌤포트는 로컬 DB를 기반으로 동작하므로 백업 기능이 중요하다.

주요 기능:

- 현재 데이터 백업 내보내기
- 기존 백업 가져오기
- 가져오기 실패 시 롤백을 위한 안전 처리

운영 팁:

- 학기별 또는 월별로 백업 파일을 별도 저장해두면 PC 변경이나 오류 상황에서 복구가 쉽다.
- 학생 개인정보가 포함된 백업 파일은 외부 공유를 금지한다.

## 26. 맥북 개발/실행 환경

현재 프로젝트는 macOS에서 Electron 앱으로 실행할 수 있다.

실행 방식:

- 프로젝트 위치: `~/Desktop/teacher-app`
- 개발 실행 명령: `npm start`
- 더블클릭 실행 파일: `~/Desktop/teacher-app.command`

로컬 AI 관련 경로:

- 모델 폴더: `~/Library/Application Support/teacher-app/ai-models`
- 실행 엔진 폴더: `~/Library/Application Support/teacher-app/ai-runtime`
- 기본 Ollama 모델: `qwen2.5:3b`

## 27. 쌤포트가 해결하려는 문제

쌤포트는 단순한 기록 앱이 아니라 교사의 업무 흐름을 줄이는 것을 목표로 한다.

해결하려는 문제:

- 업무 정보가 여러 파일과 사이트에 흩어지는 문제
- 상담 기록이 후속 조치로 이어지지 않는 문제
- 학생별 출결, 상담, 관찰, 평가 자료를 한 번에 보기 어려운 문제
- 공문과 메신저 속 업무를 사람이 직접 할일로 정리해야 하는 문제
- 시간표, 급식, 학사일정 확인을 매번 따로 해야 하는 문제
- 성적과 자격증을 바탕으로 고졸 채용 가능성을 판단하기 어려운 문제
- 학생 개인정보를 외부 AI나 클라우드에 보내면 안 되는 문제

## 28. 최종 지향점

쌤포트의 최종 지향점은 “교사의 하루를 정리해주는 안전한 업무 허브”다.

교사는 쌤포트에서 오늘의 할일을 보고, 학생 기록을 남기고, 상담 후속 조치를 정리하고, 공문을 작성하고, 취업지도 근거를 확인할 수 있다. AI는 교사를 대신해 판단하는 존재가 아니라, 반복적인 정리와 문장 작성, 근거 확인을 도와 교사가 학생에게 더 많은 시간을 쓸 수 있게 하는 보조 도구로 사용된다.

쌤포트는 편리함보다 학생 정보 보호를 우선한다. 공식 근거가 없는 추천은 하지 않고, 민감한 학생 데이터는 로컬에서 관리하며, 교사의 최종 판단과 학교 절차를 존중하는 방향으로 발전해야 한다.
